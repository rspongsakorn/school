# Design Spec: คู่มือการใช้งานระบบ (User Manual)

**Date:** 2026-05-30
**Status:** Approved
**System:** ระบบบริหารค่าเล่าเรียน — โรงเรียนบัวใหญ่วิทยา (อ.บัวใหญ่ จ.นครราชสีมา)

---

## 1. Goal

จัดทำคู่มือการใช้งานภาษาไทยแบบละเอียด ทีละขั้นตอน พร้อมภาพประกอบและหน้าปก
ครอบคลุมทุกบทบาท (admin / finance / teacher)

## 2. Deliverable

- **รูปแบบ:** Microsoft Word (`.docx`)
- **ชื่อไฟล์:** `คู่มือการใช้งานระบบ-โรงเรียนบัวใหญ่วิทยา.docx`
- **ภาษา:** ไทย, ฟอนต์ตระกูล Noto Sans Thai / Sarabun
- มีหน้าปก, สารบัญ, เลขหน้า, หัวข้อแบบมีลำดับ

## 3. ภาพประกอบ (mockups)

- ไม่ใช้สกรีนช็อตจริง (backend ต่อไม่ติด: คีย์ Supabase ว่าง, ไม่มี Docker)
- สร้างหน้า HTML จำลองให้ตรงกับ UI จริง โดยอ้างอิงจากโค้ด:
  - sidebar กว้าง 260px, โลโก้ + "โรงเรียนบัวใหญ่วิทยา"
  - สีหลักน้ำเงิน `#1B6CA8`, พื้น `#F8FAFC`, ปุ่ม CTA การเงินเหลืองอำพัน `#D97706`
  - ป้ายเมนู/ไอคอนตรงกับ `app-sidebar.tsx`
  - ข้อมูลตัวอย่างสมจริง (ชื่อนักเรียนไทย, จำนวนเงินบาท)
- เปิดใน browser → ถ่ายภาพ PNG → ฝังในไฟล์ Word
- ประมาณ 15–20 ภาพ

## 4. โครงสร้างเนื้อหา

1. หน้าปก — โลโก้, ชื่อระบบ, เวอร์ชัน, วันที่
2. บทนำ — ระบบคืออะไร, ตารางสิทธิ์ 3 บทบาท, ความต้องการเบื้องต้น
3. เริ่มต้นใช้งาน — เข้าสู่ระบบ, ส่วนประกอบหน้าจอ, ตัวเลือกปี/ภาคเรียน, ออกจากระบบ
4. ข้อมูลพื้นฐาน — ภาพรวม(แดชบอร์ด), ปีการศึกษา, นักเรียน(+นำเข้า CSV), ลงทะเบียน
5. งานการเงิน — ตั้งค่าค่าธรรมเนียม, ประเภทใบเสร็จ, ใบแจ้งชำระ, บันทึกการจ่าย, พิมพ์/พิมพ์ซ้ำใบเสร็จ, ยกเลิกใบเสร็จ
6. รายงาน — รายรับรายวัน, ค้างชำระ, สถิติการเก็บ, รายบุคคล
7. ผู้ดูแลระบบ — จัดการผู้ใช้/บทบาท
8. คำถามที่พบบ่อย & แก้ปัญหาเบื้องต้น

แต่ละหัวข้อย่อย: คำอธิบาย → ขั้นตอนมีหมายเลข → ภาพประกอบ → กล่องเคล็ดลับ/ข้อควรระวัง

## 5. Navigation reference (จากโค้ดจริง)

- **ข้อมูลพื้นฐาน:** ภาพรวม, ปีการศึกษา, นักเรียน, ลงทะเบียน
- **การเงิน:** ตั้งค่าค่าธรรมเนียม, ประเภทใบเสร็จ, ใบแจ้งชำระ, บันทึกการจ่าย, รายรับรายวัน, รายงานค้างชำระ, สถิติการเก็บ, รายบุคคล
- **ระบบ (admin):** จัดการผู้ใช้
- **teacher** เห็นเฉพาะ: รายงานค้างชำระ, สถิติการเก็บ, รายบุคคล
